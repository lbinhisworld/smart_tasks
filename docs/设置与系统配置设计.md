# 设置与系统配置 — 业务逻辑设计

本文档说明**系统配置**的入口、Tab 结构及与**全应用**的协作。配置通过顶部导航 **齿轮** 打开 **`AppConfigModal`**（`src/components/AppConfigModal.tsx`），不单独占用路由；与**当前视角、任务、大模型、Smartsheet、看板助手**等**强相关**。

---

## 1. 入口与壳层

- **入口**：`AppShell` 中 **齿轮按钮** `setConfigModalOpen(true)`，**标题/aria** 含「大模型 Key、部门架构」等。  
- **与路由无绑定**：`App` 的 `page` 可为 board/reports/… 任意，配置模态**叠在壳上**。  
- **当前视角**（`role-bar` 中的「当前视角」）**不**在设置弹窗里改分类名，而由 **`TaskContext.user.perspective` + 部门架构**生成下拉（见 `buildPerspectiveOptions`）；架构变化时 **`ORG_STRUCTURE_CHANGED_EVENT`** 触发 **重算**可选项，若当前项被删则**回退到第一项**（`useEffect` 在 `AppShell`）。  

---

## 2. 配置弹窗 Tab 一览

以下为 `ConfigTab` 类型所涵盖能力（**以实际代码为准，名称在 UI 中可能为中文**）：

| 逻辑 Tab id | 核心内容 |
|-------------|----------|
| **llm** | 大模型 **API Key** 存取（`getStoredDeepseekApiKey` / `set` / `clear`）、与 `llmExtract` 中 **readLlmEnv** 统一；**LLM 配置变化** 通过 **`LLM_CONFIG_CHANGED_EVENT`** 通知**报告/数据中台**等**重算环境**。 |
| **llm_logs** | 近期 **LLM 调用日志**（`llmCallLog`），可**清空**、展示模型/耗时/tokens/模式等。 |
| **org** | **部门架构**多行文本：解析/保存（`parseOrgStructureUserInput`、`orgStructureStorage` 的 get/set），驱动 **视角列表** 与**任务/报告可见性**（`leaderPerspective`）。 |
| **smartsheet** | **企业微信/智能表 Webhook** 多行、用途标签（`smartsheetWebhooksStorage`）；**任务字段**与 Smartsheet 列 **field id** 映射（`smartsheetTaskFieldIdsStorage`）、**恢复默认**。 |
| **history_md** | 助手/对话 **历史 Markdown** 的读取、下载、保存到本地文件（`assistantHistoryMd`），便于审计或备份。 |
| **core_memory_md** | **核心记忆**正文（`coreMemoryStorage` / `homeAssistantPrompt` 相关 get/set + 下载），与**看板 AI** 人格与长期背景绑定。 |
| **chat_skill_md** | 助手 **技能** Markdown 与**校验**（`aiChatSkillStore`、`validateAiChatSkillMarkdownShape`），控制工具/指令形态。 |
| **chat_skill_update_md** | **技能修订**区（`aiChatSkillRevision`），与技能主文档**分工**为「增量更新」类说明。 |

---

## 3. 与全应用的依赖关系

- **大模型**  
  - **报告管理**、**数据中台** JSON 清洗、**看板 Home AI** 等均通过 **`llmExtract`** 读同一环境。  
- **部门架构**  
  - 影响 **视角**、**任务可见**、**报告历史可见**、任务表单里**组织**合法性（与 `orgStructureContainsDepartment` 等配合）。  
- **Smartsheet**  
  - **任务管理** 中推送/重推、Webhook 行配置；**字段 id** 与「任务行 → 表列」**一一对应**方可稳定更新。  
- **核心记忆与技能**  
  - 主要消费方 **`HomeAiChatPanel`** 与用户模式管线（`userModeStreamDisplay`、`homeAssistantUserModePipeline` 等）中的 prompt 装配。  

---

## 4. 持久化与事件（摘要）

| 能力 | 存储或事件 |
|------|------------|
| API Key | `llmExtract` 内 localStorage/约定 key |
| 部门架构 | `orgStructureStorage` + `ORG_STRUCTURE_CHANGED_EVENT` |
| Webhook / Field ids | `smartsheetWebhooksStorage`、`smartsheetTaskFieldIdsStorage` |
| 核心记忆 | `setCoreMemoryText` 等（见 `homeAssistantPrompt` / `coreMemoryStorage`） |
| 技能 md | `aiChatSkillStore`、`aiChatSkillRevision` |
| 助手历史 md | `assistantHistoryMd` |
| LLM 配置变更 | `LLM_CONFIG_CHANGED_EVENT`（报告、DataSync 等订阅） |
| 日志变更 | `LLM_CALL_LOG_CHANGED_EVENT`（配置页刷新日志区） |
