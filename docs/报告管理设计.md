# 报告管理 — 业务逻辑设计

本文档说明「报告管理」页：生产报告**上传/解析/预览/写入任务与历史**、**提取历史**、与看板/助手/数据中枢的**事件协作**。主代码：`src/components/ReportManagement.tsx` 及 `src/utils/` 下报告提取、历史、计划任务、引文、LLM 等模块。

---

## 1. 功能定位

- **双 Tab**：**报告提取**（主流程：文件 → 文本/JSON → 大模型按 schema 抽数 → 预览与各类动作）与**提取历史**（按时间/分公司等浏览、高亮、删除、导入导出）。  
- **与路由**：`App` 中报告模块 **lazy 加载**；`OPEN_REPORTS_PAGE_EVENT` 可**从看板等**将路由切到报告页。  
- **多页同挂**：`App` 对 `board` / `reports` / `tasks` 等使用 **`display: none` 同挂**，使报告在「未显示」时仍可被**助手事件**驱动解析（见 `App.tsx` 注释与 `ASSISTANT_UI_ACTION_EVENT` 处理：助手**不** `navigate` 切整页，用自定义事件**焦点/触解析**）。  

---

## 2. 报告提取主流程（概念）

1. **输入**：原文件经 `extractTextFromFile` 等得到正文；**提取日期**优先文首/规则（`extractPrimaryReportDateFromPlainText`），与 `formatExtractionDate` / `normalizeProductionReportJson` 对齐。  
2. **LLM 调用**：`llmExtract` 中 `callProductionReportExtraction` 等，依赖 **readLlmEnv**（与设置里 API Key/模型一致），返回**结构化 JSON** 再规范化。  
3. **落历史**：`appendExtractionHistory` 写入**提取历史**；含 **LlmCallStats、定量指标引文** `quantitativeMetricCitations` 等。  
4. **任务侧能力**（典型）：  
   - 从解析结果/分公司段落 **更新任务进度**（`appendTaskProgressEntry`、`taskMatchesReportCompany` 等）。  
   - 从 **6.1/6.2** 等区块 **生成「日报计划任务」** 预览行、再批量生成**真实任务**（`generateTaskFromDailyPlanRow`、`llmSplitDailyPlanItemsFromSections` 等）。  
5. **草稿与侧栏状态**：`reportExtractionPreviewDraft`、`reportSidePanelsDraft` 等**本地草稿**，防刷新丢填写。  
6. **看板/助手联动**：`EXTRACTION_FOCUS_STORAGE_KEY` + `sessionStorage` + `REPORT_FOCUS_EXTRACTION_EVENT` / `consumeExtractionFocusFromStorage`，实现「**跳转原文** / 高亮到某次提取」。

---

## 3. 提取历史

- **存储**：`extractionHistoryStorage`：加载、追加、删除、**导入 JSON** 等。  
- **分组/时间线**：`extractionHistoryGroup`（`buildTimelineGroups` 等）给列表与**报告看板日期轴**一致。  
- **视角过滤**：`extractionHistoryVisibleForPerspective` 与**任务/看板**同一 `perspective` 概念。  
- **标题/展示**：`buildExtractionHistoryTitle` 等。  
- **类型**：`ExtractionHistoryItem`、计划任务行、引用等见 `src/types/extractionHistory.ts`。

---

## 4. 报告看板与引用

- 首页**报告** Tab（`ReportDashboardTab`）用历史数据**聚合产量日看板**（`productionDashboardMetrics`、`reportCompanyDailySlices` 等），点击指标打开 **`ReportCitationDrawer`**（**原文片段 + 多段高亮**）。  
- `reportCitation`：**跳转原文** `requestJumpToExtractionHistory` 与**报告页**消费 `sessionStorage` 配合。

## 7. 动态记忆与计划上下文

- **报告类动态记忆**：`reportDynamicMemory`（如随历史**重建/同步**）

## 8. 持久化与事件（摘要）

| 数据 | 说明 |
|------|------|
| 提取历史 | `extractionHistoryStorage` |
| 预览/侧栏草稿 | `reportExtractionPreviewDraft`、`reportSidePanelsDraft` |
| 原始包下载 | `downloadOriginalReportsJson` 等 |
| LLM 环境 | `llmExtract`、与**设置**同步 `LLM_CONFIG_CHANGED_EVENT` |

| 事件（节选） | 用途 |
|--------------|------|
| `OPEN_REPORTS_PAGE_EVENT` | 切到报告路由 |
| `REPORT_FOCUS_EXTRACTION_EVENT` | 聚焦提取区 |
| `ASSISTANT_REPORT_PARSE_EVENT` / `ASSISTANT_SET_REPORT_MANUAL_TEXT_EVENT` | 助手触解析/填文 |
| `dispatchAssistantReportChainDone` 等 | 链式与助手回传 |
