# 数据中台 — 业务逻辑设计

本文档说明「**数据中台**」路由下 **`DataSync`** 页的能力：外部**数据源/接口**配置、**cURL 导入**、**联调测试**、**响应体**在 **业务行 / 原始 JSON / 大模型清洗后 JSON** 三视图下浏览与**字段展示配置**，以及**单元格详情抽屉**。主代码：`src/components/DataSync.tsx`；状态与行为见 `externalApiStorage`、`dataSyncResponseStorage`、`dataHubJsonCleaning` 等。

---

## 1. 功能定位

- **与「销售预测/报告管理」中的 DataHub 关系**：**同一套** 接口配置 与**最近一次响应** 可被 `loadDataHubState` / `loadDataSyncLastBody` 等在**报告、计划、助手**中**读取**，实现「**一次配置、多处复用**」。  
- **CORS**：浏览器直连接口可能受 CORS 限制，文档注释中说明需 **Vite 代理**或**后端转发**。  

---

## 2. 信息架构

- **按数据源（平台）分组**：`DataPlatform` 下挂多个 `ExternalApiProfile`（`types/externalApiProfile.ts`），每一配置含 URL、方法、头、**清洗规则**说明等。  
- **主 Tab**  
  - **基础/接口**（`basic`）：**cURL 粘贴解析**（`parseCurl`）→ 快速填充配置；**测试请求**、耗时与 HTTP 状态、错误、**截断长响应**预览。  
  - **数据**（`data`）：在选定接口上展示**三类子 Tab**。  

---

## 3. 数据三子 Tab

| 子 Tab | 含义 |
|--------|------|
| **业务数据** | 将 **原始 JSON** 用 **启发式/固定路径** 抽为**行表** `extractBusinessRowsFromJson`；支持列筛选、搜索、**可见列**与**拖拽列序**（`buildDataHubFieldPickerOrder`）。 |
| **原始 JSON** | 上次测试返回的**原文**（或从存储恢复），便于对照。 |
| **清洗后的 JSON** | 用户**自然语言规则** + 调用大模型 `callLlmChatText`（**独立系统提示**与 `dataHubJsonCleaning` 中约定），输出**合规格式化** JSON 字符串；**会话级** 还可写入 `session` 前缀的 **DATA_HUB_CLEANED_JSON_***（`dataHubCleanedJsonStorage`），供报告/解析链路 **按 key 取**。 |

---

## 4. 与 LLM/配置

- **大模型环境**与**设置**中 API Key/模型**同源**（`readLlmEnv`、`LLM_CONFIG_CHANGED_EVENT`），修改后数据中台**无需**另存 key。  
- 清洗有 **max tokens** 等限制常量（`DATA_HUB_JSON_CLEANING_MAX_COMPLETION_TOKENS` 等）避免超长响应把上下文撑满。  

---

## 5. 持久化

| 内容 | 存储 |
|------|------|
| 平台+接口配置列表 | `externalApiStorage`（`saveDataHubState` / `loadDataHubState`） |
| 最近测试响应体等 | `dataSyncResponseStorage`（如 `saveDataSyncLastBody`） |
| 清洗结果（会话，键前缀） | `dataHubCleanedJsonStorage` |
