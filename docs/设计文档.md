# 设计文档（首页看板与报告维度）

本文档描述首页「数据看板」中与**当前视角**相关的数据范围与 UI 文案规则，便于产品与实现对照。

## 1. 首页数据看板结构

- **任务看板**：任务列表、范围筛选、风险与分类统计等；任务可见性由 `taskVisibleForPerspective` 决定（见 `src/utils/leaderPerspective.ts`）。
- **报告看板**：`ReportDashboardTab`，展示按提取日聚合的产量 KPI 树（集团 / 分公司 / 车间）及原文引用侧栏。父组件 `Dashboard` 在「报告看板」Tab 下从本地存储加载提取历史，并按视角过滤后传入子组件。

相关入口：`src/components/Dashboard.tsx`。

## 2. 当前视角（领导视角）

- 顶部可选视角包括固定项 **「集团领导」** 以及部门架构中每一行对应的 **「{架构单元}领导」**（由 `buildPerspectiveOptions` 生成）。
- **集团领导**：组织单元为「全集团」，不对报告提取历史做分公司维度过滤。
- **分公司领导**（架构名以「分公司」结尾，如「广西分公司领导」）：报告提取历史仅保留 `pickBranchCompany(item)` 与当前分公司一致的数据。
- **非分公司职能部门领导**：报告提取历史仍看**全部**记录（与集团领导在报告维度上的数据全集一致），与 `extractionHistoryVisibleForPerspective` 行为一致。

过滤函数：`extractionHistoryVisibleForPerspective`（`src/utils/leaderPerspective.ts`）。  
`Dashboard` 使用：`filteredReportHistory = reportHistory.filter(h => extractionHistoryVisibleForPerspective(h, user.perspective))`。

## 3. 报告看板：一级「当日产量指标」括号内说明

一级汇总卡片标题格式为：

> **当日产量指标（{scopeLabel}）**

`scopeLabel` 须与**当前视角下实际参与聚合的提取历史范围**一致（即与传入 `ReportDashboardTab` 的 `history` 一致），规则如下：

| 当前视角 | `scopeLabel`（括号内） |
|----------|------------------------|
| 集团领导 | 全部分公司汇总 |
| 非分公司职能部门领导（可看全量报告历史） | 全部分公司汇总 |
| 分公司领导（如「广西分公司领导」） | **{分公司名}汇总**（如「广西分公司汇总」） |

实现：`reportDashboardLevel1ScopeLabel(perspective)`（`src/utils/leaderPerspective.ts`）。  
展示：`ReportDashboardTab` 接收 `perspective`，标题为 `当日产量指标（${level1ScopeLabel}）`；`Dashboard` 传入 `perspective={user.perspective}`（`src/components/ReportDashboardTab.tsx`、`src/components/Dashboard.tsx`）。

## 4. 与其它文案的关系

- 原文引用侧栏等场景若仍使用「全集团汇总」等固定文案，属于**引用展示维度**的命名，可与上述一级卡片括号说明独立演进；若产品要求全文案统一，需单独评审后改 `ReportCitationDrawer` / `reportCitation` 等相关逻辑。
